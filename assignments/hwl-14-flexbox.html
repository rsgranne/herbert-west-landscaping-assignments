We’re so close now. Basically, we need to take care of problem #2 above — different widths! — which might just fix problem #4 too. Will it? Stay tuned to find out!

To do this, we need to select the <section>s that are the direct children of .row, which means a selector of .row > section. In order to understand the declarations we will need, we have to dive into the details of flexbox and how to size flex items like <section>.

By default, flex items are sized to their content. Longer content means a longer flex item. That’s why some of the <section>s are wider than others: more content. That most definitely is not what we want; instead, we want all of the flex items to be the same width. To understand how to do that, we need to look at 3 key flex-related properties having to do with sizing flex items:

  • flex-shrink: Allows flex items to shrink if necessary to fit into their flex container. The default is 1, which means that shrink is allowed.
  • flex-grow: Allows flex items to grow if necessary to fit into their flex container. The default is 0, which means that growing is not allowed.
  • flex-basis: Defines initial main size of flex items before changes caused by flex-grow or flex-shrink are applied. The default is auto, which tells rendering engines to size flex items based on their content. Aha! That is why flex items are sized to fit their content by default! Clearly we do not want that if we want all of our flex items to have the same width.

By default, all flex items can shrink, which translates to a declaration of flex-shrink: 1. Since that is the default, we don’t need to add that. In fact, we want our flex items to be able to shrink if needed. There’s a reason that it’s a default — it’s normally what you want.

However, we do want our flex items to grow as needed. By default, flex items do not grow (flex-grow: 0). We need to change that behavior by using flex-grow: 1, which tells the rendering engine to make our flex items wider if necessary.

Hmmm. Adding that didn’t fix the differing widths. That’s because of flex-basis, which has the following possible values:

  • auto: The default, which sizes based on content. We already know we don’t want this one.
  • max-content: Sizes flex items to the widest they need to be based on the maximum size of their content, which means they can expand as much as needed (text will still wrap though). We definitely do not want that.
  • min-content: Sizes flex items to the narrowest they need to be, based on the widest content, in other words, the longest word or image! Since we are using flex-grow: 1, min-content doesn’t have a lot of effect, but if you changed flex-grow to the default of 0 and then used flex-basis: min-content, you’d see something pretty silly. So yeah, we don’t want this one.
  • <width>: That’s not an element (there isn’t any), but is instead representing any allowed width in CSS, like 1rem or 20px. First off, using <width> sets a fixed size, which kind of goes against the whole responsive thing, because the flex items can’t resize as the viewport width changes. In fact, our flex items could either overflow or truncate if the available space in the flex container is not enough to accommodate their fixed size. We sure don’t want that.
  • 0: Do not base the initial size of flex items on content, so growing and shrinking occur without content in mind. Hmmm. That one sounds promising…

OK, let’s think about this. In order to get all of our cards the same size…

  • We want our flex items to be able to shrink as needed, so flex-shrink: 1 (which is the default)
  • We want our flex items to be able to grow as needed, so flex-grow: 1 (the opposite of the default)
  • We want the initial size of our flex to ignore the size of content, so shrinking and growing can occur without content causing an issue. That means flex-basis: 0.

So add those declarations to .row > section. Save it and look at the results using Live Preview. Great Googly-Moogly, they’re all the same width! And the images are all the same size too!

Before we leave our cards, I do want to point out one thing to you. Earlier we specified flex-shrink, flex-grow, & flex-basis to size our cards. Actually, there is shorthand that you can use instead: flex. Yep. You can just use flex and that way you don’t need to specify flex-shrink, flex-grow, & flex-basis.

If you use flex, the values have to be in this order: flex-grow flex-shrink flex-basis. Therefore, what we did earlier could instead be expressed as: 

flex: 1 1 0

However, there’s an even shorter way to do that (shorthand for shorthand?!). If you simply follow flex with 1, then flex uses that number for the first position, flex-grow. The second 1 is already the default for flex-shrink. And 0 for flex-basis is assumed in this case. Therefore, each of the following 3 are equivalent.

All property-value pairs: 

flex-shrink: 1;
flex-grow: 1;
flex-basis: 0;

Shorthand, specifying all values: 

flex: 1 1 0;

Shorthand with one number that specifies the value of flex-grow only: 

flex: 1;

Me, I like to use the last one. If you wanted, you could use all 3 in main.css with comments for each one, like this: 

/* All property-value pairs: */
flex-shrink: 1;
flex-grow: 1;
flex-basis: 0;
/* Shorthand, specifying all values: */
flex: 1 1 0;
/* Shorthand with one number that specifies the value of flex-grow only: */
flex: 1;

The last one is the one that will be used, of course.
