Reasoning

Up until now you have copied the <footer> from webpage to webpage, which is tedious. On top of that, if you needed to make a change, you had to make that change to the <footer> on every page, which is also tedious & error-prone. The same thing has been true, by and large, for the <header>.

What if there was a better, less tedious way to work with the <footer>? Ideally, you would only have to update the <footer> in one place, & it would automatically update everywhere that <footer> was used. That would be great!

In this assignment, you will learn how to do just that. You will remove the <footer> section from every current webpage & will instead put the <footer> code into a single file named footer.html (this will not be a complete HTML file; instead, it will only contain the content of the <footer> in it). You will then use JavaScript to automatically place the contents of footer.html into each webpage where the <footer> used to go. When a user loads the webpage, the contents of footer.html will appear. If you need to change the <footer>, you simply update it once in footer.html, & the next time someone loads a webpage, your changes will appear. That’s cool!

A few notes:

Yes, we can use JavaScript to insert other sections that are common to all webpages on the site. The nice thing about <footer> is that it is always exactly the same on every webpage on the site. That makes the JavaScript pretty simple.

However, the second half of this assignment uses JavaScript to automatically place the breadcrumb navigation on every page. This is more complicated, because while every page has a breadcrumb, they are all different! Fortunately, JavaScript is a pretty powerful programming language & can automatically generate & insert the code for the breadcrumb on every webpage. As a result, the JavaScript is quite a bit more complicated. But that’s OK, since the purpose of this assignment is not to teach you JavaScript; instead, it’s to teach you how to use JavaScript.

And yes, some of you who know PHP are probably thinking that we could use that programming language to do the same things we are using JavaScript for in this assignment. However, I decided against that for two reasons:

  1. PHP is a huge subject that is way beyond the scope of this course.
  2. PHP requires that you use a different path to footer.html depending upon the page. In other words, sometimes you link to includes/footer.html, sometimes to ../includes/footer.html, & sometimes to ../../includes/footer.html. After hours of research & testing, it appears that there is no easy way to simply use /includes/footer.html every single time with PHP. You can, however, do exactly that with JavaScript, & since that fits in with what we’ve been using with every other link in Herbert West Landscaping, that’s what we’ll use.

<noscript>

Before we leave, I have to explain something you see in the code we added to every page: <noscript>.

In order to see the footer & its content, the user must have JavaScript enabled. The vast majority of users never disable JavaScript, but a few do. For those folks, they just won’t see a footer at all, nor will they even know that a footer should be there but isn’t. This is not good. Fortunately, HTML provides a way to let people see something if JavaScript is disabled: the <noscript> tag. That’s why we added it above.

By default, <noscript> will be right against the bottom edge of the browser window, which makes it hard to see. Let’s fix that by updating our CSS so that our message has some space below it, so add this to main.css 

body > noscript:last-of-type {
  padding-bottom: 2rem;
}

To understand :last-of-type in that CSS, you first need to learn about :last-child. As I will explain below when you use it with the breadcrumb, the :last-child pseudo-class selects the last child of a parent element (hence the name!). Let’s say you had a <div> that had 5 children, all siblings: an <h2>, then two <p> tags, then another <h2>, & finally another <p>. 

<div>
  <h2>Nyarlathotep</h2>
  <p>Abdul Alhazred</p>
  <p>Randolph Carter</p>
  <h2>Hastur</h2>
  <p>The King in Yellow</p>
<div>

Now let’s say I had this selector in my CSS: div > *:last-child. That tells the rendering engine to match the element that is the last child of the <div>, which in this case would be the <p> around The King in Yellow.

What if I added a <ul> after that last <p>? Then :last-child would now match the <ul> because that would now be the last child of <div>. Got it?

Now :last-child hopefully makes sense, & you will be using it shorly below, so that will help you understand it even more. But what is :last-of-type?

What if I want to target the last <h2> in the <div> & put a special border on top of it to distinguish that last implicit section in the <div>? And what if someone editing this page might put another <h2> & a few <p>s after the current last <p> in the <div>, like this: 

<div>
  <h2>Nyarlathotep</h2>
  <p>Abdul Alhazred</p>
  <p>Randolph Carter</p>
  <h2>Hastur</h2>
  <p>The King in Yellow</p>
  <h2>Cthulhu</h2>
  <p>Deep Ones</p>
<div>

I could put a class on the last <h2>, but every time another <h2> was added or removed at the end, I’d have to remove & re-apply the class, & there is no way I’m going to do that. So how do I target not the last child but instead the last <h2> inside the <div>? With :last-of-type, specifically, div > h2:last-of-type.

If div > *:last-child specifies the last child element of the <div>, no matter what element that may be, then div h2:last-of-type targets only the last <h2> that’s a child of <div>.

In other words, div > *:last-child tells the browser to count every element that’s a child of <div> & then choose the last one. However, div > h2:last-of-type tells the rendering engine to count only the <h2>s inside the <div> & then select the last one of those.

After all that, it should be clear why we used body > noscript:last-of-type above as our selector. I don’t want to use body > *:last-child, because I don’t know what the last child might be, & therefore I might not select the <noscript>. I don’t want to just use noscript:last-of-type, because that might select a <noscript> that is not at the very end of the webpage. Instead, I use body > noscript:last-of-type so that I’m sure to only target the very last <noscript> on the page, the one right before </body>.

To start with, let’s make this ordered list look a lot less like an ordered list. We are going to re-use the CSS we used on the social media icons in the <footer>, so put this in main.css: 

.breadcrumb {
  list-style-type: none;
  padding: 0;
  margin: 0;
  display: flex;
}

“Hey, Scott”, you may be asking, “We’re using this .breadcrumb class, but I don’t see where it is in the HTML. Where is it? Huh? Huh? Where?” First I’m going to ask you to stop being so insistent, calm down, & please back up, because you are in my space. Next, I’ll let you know that the JavaScript is writing out the <ul> that we are styling, & part of that process is adding the class of .breadcrumb (if you don’t believe me, look at line 29 of breadcrumb.js & you will see it there).

With that finished, let’s add this very complicated looking CSS: 

.breadcrumb li:not(:last-child)::after {
  content: "\00A0➤\00A0";
}

In short, li:not(:last-child)::after tells the rendering engine to insert \00A0➤\00A0 after every <li>, but not after the one that is the last child of its parent. To understand exactly how it means that, read on.

First let’s look at the declaration: content: "\00A0➤\00A0";. content tells the rendering engine to insert whatever its value is. 00A0 is the Unicode number for a non-breaking space (in HTML we’d use &nbsp, remember?). To insert Unicode ID numbers in CSS, you preface them with a \ (a backslash). In the middle of the value is ➤, a Unicode character identified as “Black Right Arrowhead”. Therefore, this declaration tells the rendering engine to insert a non-breaking space, then a ➤, & then a non-breaking space.

Now let’s examine the selector.

.breadcrumb li is easy: select every <li> that is a descendant of a <breadcrumb>. It’s everything after the li that’s a wee bit complicated.

::after is a pseudo-element (you can tell by the :: in front of it). If it simply said li::after, that would tell the rendering engine to insert the value of the content property (in this case, \00A0➤\00A0) after every <li>.

Remember that our breadcrumb looks like this:

Home / Products / Fertilizers

If the selector was simply .breadcrumb li::after, it would now look like this:

Home ➤ Products ➤ Fertilizers ➤

Hmmm. The first two uses of ➤ are fine, but we don’t want the one at the end! How do we tell the rendering engine to insert ➤ after every <li> except the last one? That’s where :not(:last-child) comes in.

:last-child is a pseudo-class (you can tell by the : in front of it) that selects the last child of a parent element. If our selector was .breadcrumb li:last-child, that would target the last <li> that was the child of a parent element, if that <li> was a descendant of .breadcrumb. Say we had this HTML: 

<div class="breadcrumb">
  <ul>
    <li>Cthulhu</li>
    <li>Nyarlathotep</li>
    <li>Shoggoth</li>
  </ul>
</div>

In this case, .breadcrumb li:last-child would select the <li> around Shoggoth, because that <li> is the last child of its parent (here, <ul>), & that <li> is a descendant of an element with a class of breadcrumb.

Going back to Home ➤ Products ➤ Fertilizers ➤, we want to insert a ➤ after every <li> except the last child. That’s where :not(:last-child) comes in.

:not() is another pseudo-class that let’s you specify DOM objects you do not want to select. You specify what you do not want to target by putting that selector inside the () after :not. In other words, li:not(:last-child) tells the rendering element that you want to select every <li> except the <li> that is the last child of a parent.

Putting it all together, li:not(:last-child)::after specifies that you want to insert \00A0➤\00A0 after every <li>, but not after the one that is the last child of its parent (and of course, those <li> elements have to be the descendant of an element with a class of breadcrumb).

Thus, given a CSS selector that looks like .breadcrumb li:not(:last-child)::after, our breadcrumb will now look like this:

Home ➤ Products ➤ Fertilizers

The selector told the rendering engine to insert \00A0➤\00A0 after each <li> (here, Home & Products), but not after the last child <li>, which here would be Fertilizers. Exactly what we want!

Hopefully you can now see how specifically CSS let’s you target selectors, & you learned a bit more about pseudo-classes & pseudo-elements.
