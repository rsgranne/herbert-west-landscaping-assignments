Font embedding

In this assignment we will practice embedding fonts, but then we are going to comment it out after making sure it works. Why? Because we are going to use the system fonts that Bootstrap uses.

Here are the steps we will follow:

  1. Select a heading font & body font at Google Fonts
  2. Go to google-webfonts-helper
  3. Select the fonts, charsets, styles, formats, & folder prefix
  4. Download the fonts
  5. Copy the CSS
  6. Move the fonts into your project’s fonts folder
  7. Edit the CSS
  8. Test

Here we go!

Google Fonts

Go to Google Fonts. I’m going to suggest fonts, but you can always choose your own. One of the easiest design suggestions is to choose a serif font for headings & a sans-serif font for the body, or vice-versa. We are going to go with the former.

First we need a serif heading font. All headings are bold, so we need a heading font with a weight of 700. We do not need a regular weight font of 400, however, because that weight is never used in headings. Likewise, we probably do not need an italic style, because that is rarely used in headings. However, if you think you might be using italic text in your headings — like book or movie titles, for instance — then you would need an italic style. In our case, let’s go ahead & assume we would need italics so we can practice it.

In the top left of Google Fonts is a dropdown labeled Categories. Click on it and then uncheck everything except Serif. That leaves around 300 fonts to choose from. I’m going to pick something that will make our changes painfully obvious, so let’s start with Slabo (go ahead & click on that link & take a look).

Wait a second! Slabo only has 1 style, & that is Regular. We can’t use it for a heading font. Let’s try again.

OK, how about Libre Baskerville (click on that link too)? That has a 700 style, but it doesn’t have a 700 italic, so it’s out as well. Back to searching.

Aha! How about Rokkit (take a look)? That one has 18 styles (100–900, as well as italics for every weight), so it will definitely have 700 & 700 italic, and sure enough it does. Whew!

That is all we are going to do on that page. We are not going to click on anything else, because we are not going to embed fonts the Google way (if you don’t recall the problems with the Google way, go look in the CSS Typography Condensed slides for the Embedding section and start reading there).

Now for a body font. Because we chose a serif heading font, we are going to select a sans-serif body font. Further, because this is a body font, we will need one that supports weight of 400 and 700, as well as 400 italic. I’m not going to look for 700 italic, because you should avoid using both italics and bold together, as that is gauche.

“But Scott,” you protest, “we just chose a 700 italic for headings!” Yes we did, because we felt like we had to, given that all headings are bold to begin with, and sometimes we might have titles as headings. But if we have a choice, don’t combine italics with bold, as that is just too much emphasis. It’s like wearing a belt and suspenders — one is enough!

Back at Google Fonts, click on the Categories dropdown and uncheck everything except Sans Serif, which will leave over 300 fonts to choose from.

How abouuuuuuut… Oswald (click on that link)? Hmmm. No. First of all, although it has 400 & 700, there are no italics. Second, it’s a condensed typeface. A condensed typeface is designed to take up less horizontal space than regular typefaces by reducing the width of its characters while maintaining its height and stroke thickness. Now, I love condensed typefaces for headings, but certainly not for the main reading font on a webpage! So Oswald is out.

Let’s take a gander at Raleway (click it!). It has 9 weights & corresponding italics, so that will work. Yes, it’s very stylized (that capital W!), but I’m purposely picking very noticeable fonts so that when you test your code using Live Preview you will definitely be able to see whether or not it has worked.

We now have our choices: Rokkit 700 & 700 italic for headings, and Raleway 400, 400 italic, & 700 for the body. On to google-webfonts-helper!

google-webfonts-helper

Head to google-webfonts-helper in your browser. First, headings.

In the search box in the upper left corner, type in Rokkit, which will filter the list to our desired typeface. Click on it and the main content of the page should change to show you Rokkit.

Step 1 is “Select charsets”, & we have 3 choices: latin (which is already checked), latin-ext, and vietnamese. “Latin” means western European, which includes English, French, Spanish, and the like. Do we need “latin-ext” (the “ext” is short for “extended”)?

According to Wikipedia, Latin Extended-A includes alphabets for Afrikaans, Catalan, Croatian, Czech, Esperanto, Greenlandic, Hungarian, Kashubian, Kurdish, Latin, Latvian, Lithuanian, Maltese, Northern Sami, Polish, Romanian, Serbian, Slovak, Slovene, Sorbian, Turkish, & Welsh. Do we need these for this assignment? Nope. So we do not need to check “latin-ext”!

There are other, additional categories of Latin Extended that UTF-8 supports, but not Google Fonts (unless a particular web font says otherwise).

Latin Extended-B includes alphabets for Africa, Americanist, Khoisan, Pan-Nigerian, Pinyin, & Romanian. I’ll stop there, but if you’re interested, check out Latin Extended-C, D, E, F, & G.

Step 2 is “Select styles”. Notice that “regular” (which is 400) is already checked by default. We don’t want that one, but we cannot uncheck it until we have checked something else. The styles we want are 700 & 700 italic, so check those and then uncheck 400.

Step 3 is “Copy CSS”. Since we don’t care about older browsers that do not support WOFF2 (97% of all browsers support WOFF2, as of March 2024), we will select Modern Browsers instead of Legacy Support or Historic Support (go ahead & click on those if you want to see what they include).

At the bottom of step 3 it says “Customize folder prefix (optional):” and the default is set to ../fonts/. This is not a good default and it certainly is nothing like the paths we have been using, so we need to change it to /fonts/. Make sure you do this. As soon as you do you should see the code in the gray box change to conform to what we entered.

Before we copy the CSS, however, I always go ahead to step 4, “Download files”, and click on the big blue button to start the download. That will download a file named something like rokkitt-v34-latin.zip into your Downloads folder. Leave it alone for now and go back to step 3.

main.css

Time to copy that CSS, so just click anywhere in the gray box, which will select all of that code. Press ⌘C if you are on a Mac or Ctrl+C if you use Windows or Linux to copy the code.

Go to main.css and paste in that CSS at the very top of the file, before anything else. It should look like this: 

/* rokkitt-700 - latin */
@font-face {
  font-display: swap; /* Check https://developer.mozilla.org/en-US/docs/Web/CSS/@font-face/font-display for other options. */
  font-family: 'Rokkitt';
  font-style: normal;
  font-weight: 700;
  src: url('/fonts/rokkitt-v34-latin-700.woff2') format('woff2'), /* Chrome 36+, Opera 23+, Firefox 39+ */
       url('/fonts/rokkitt-v34-latin-700.woff') format('woff'); /* Chrome 5+, Firefox 3.6+, IE 9+, Safari 5.1+ */
}
/* rokkitt-700italic - latin */
@font-face {
  font-display: swap; /* Check https://developer.mozilla.org/en-US/docs/Web/CSS/@font-face/font-display for other options. */
  font-family: 'Rokkitt';
  font-style: italic;
  font-weight: 700;
  src: url('/fonts/rokkitt-v34-latin-700italic.woff2') format('woff2'), /* Chrome 36+, Opera 23+, Firefox 39+ */
       url('/fonts/rokkitt-v34-latin-700italic.woff') format('woff'); /* Chrome 5+, Firefox 3.6+, IE 9+, Safari 5.1+ */
}

There is a lot of garbage in there that we do not need, starting with the comments, which include anything from /* to */. Get rid of all that, so it should now look like this (notice I also put a blank line between the two @font-face style sets): 

@font-face {
  font-display: swap;
  font-family: 'Rokkitt';
  font-style: normal;
  font-weight: 700;
  src: url('/fonts/rokkitt-v34-latin-700.woff2') format('woff2'),
       url('/fonts/rokkitt-v34-latin-700.woff') format('woff');
}

@font-face {
  font-display: swap;
  font-family: 'Rokkitt';
  font-style: italic;
  font-weight: 700;
  src: url('/fonts/rokkitt-v34-latin-700italic.woff2') format('woff2'),
       url('/fonts/rokkitt-v34-latin-700italic.woff') format('woff');
}

There is one more thing to change: instead of a font-family name of 'Rokkitt', let’s use Heading. Why? Because that way we can change the font at any time & just change the url() parts without needing to change everywhere else in our CSS in which we refer to the font! In other words, assume we went with Rokkitt as the font-family, which would mean we would do this: 

h1, h2, h3, h4, h5, h6 {
  font-family: Rokkitt, serif;
}

Let’s say we change the font later on. Now we have to change the url() lines and the font-family name in @font-face, and then change everywhere else in our CSS that we referred to Rokkitt. Too much work!

If we use Heading instead in @font-face, we can do this: 

h1, h2, h3, h4, h5, h6 {
  font-family: Heading, serif;
}

Now if we decide to use a different font later, we just change the url() lines in @font-face and that is all. We don’t need to change the font-family in @font-face, nor do we need to change it anywhere else in our CSS. Much easier!

So if we follow my advice, we now have this in main.css (& notice we also get rid of the completely unnecessary single quotation marks around Rokkit as well): 

@font-face {
  font-display: swap;
  font-family: Heading;
  font-style: normal;
  font-weight: 700;
  src: url('/fonts/rokkitt-v34-latin-700.woff2') format('woff2'),
        url('/fonts/rokkitt-v34-latin-700.woff') format('woff');
}

@font-face {
  font-display: swap;
  font-family: Heading;
  font-style: italic;
  font-weight: 700;
  src: url('/fonts/rokkitt-v34-latin-700italic.woff2') format('woff2'),
        url('/fonts/rokkitt-v34-latin-700italic.woff') format('woff');
}

Looking good. Now let’s go down in main.css to our headings selector — which is h1, h2, h3, h4, h5, h6 — and create a new font-family line after the current one (the Cascade at work!) so that it looks like this: 

h1,
h2,
h3,
h4,
h5,
h6 {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, 'Open Sans', 'Helvetica Neue', sans-serif;
  font-family: Heading, serif;
}
