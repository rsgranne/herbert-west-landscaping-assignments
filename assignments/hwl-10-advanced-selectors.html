In this assignment you will use these new CSS properties, sorted into categories:

  • Typography
    • text-decoration: Allows you to insert or remove lines above, below, & through text. This is what browsers use to automatically put lines under <a>s. Of course, you can also use this property to remove lines from under <a>s as well. For more on this property, look for the text-decoration section in the CSS - Typography presentation.
  • Multi-column layout lets you specify a number of columns & then have content flow from the first column into the second, & from the second into the third, & so on, much like a newspaper article does when it reaches the bottom of a column & continues in the next. For more on these properties, look for the “Multi-Column Layout” section in the CSS Layout: The Visual Formatting Model presentation.
    • column-count: Specifies the number of columns you want for your content.
    • column-width: Acts as a hint to the browser of your optimal width for the columns. Depending upon the width of the columns’ container, the actual width may be wider or narrower. If you set this to auto instead of a number, the browser will determine the width based on column-count & other related CSS properties.
    • column-gap: The size of the gutter between your columns, specified using the normal CSS lengths, such as px, em, or rem.
    • column-rule-width: You can create a line, or rule, between your columns. Just as you can with border, you can set the rule’s width, style, & color. This property sets the width of the rule, specified using the normal CSS lengths.
    • column-rule-style: Sets the style of the column rule. Possibilities include the same values you saw with border-style, such as solid, dotted, or none.
    • column-rule-color: Sets the color of the rule, using the same ways to specify colors that you can use anywhere you need to do so in CSS.
    • column-rule: This acts as a shortcut for column-rule-width, column-rule-style, & column-rule-color, for example, column-rule: 2px dotted #222. Yes, this is very similar to the border shortcut. And just like border, the order of values doesn’t matter.

<footer>

We want all of the <a>s inside the <footer> to:

  • Be this color: hsl(150, 70%, 31%) (HSL is a way to specify color that is extremely useful, as opposed to hexadecimal values like #7c392e, which are impossible to understand without looking them up)
  • Have the underline removed, but show up while a user is hovering over the link

Be aware that the <a> is unique among elements because it can have 4 states, all indicated using pseudo-classes, which always start with a : (for more on pseudo-classes, look for the “Width & Height” section in the CSS Layout: The Visual Formatting Model presentation.) The 4 pseudo-classes associated with <a> are:

  • a:link, which lets you style unvisited links; the defaults are blue & underlined
  • a:visited, which lets you style visited links; the defaults are purple & underlined
  • a:hover, which lets you style links while your mouse hovers over them (obviously this doesn’t work on mobile!); the defaults are to change the cursor into a hand.
  • a:active, which lets you style links the very brief moment during which they are clicked; the defaults are red with the hand cursor.

IMPORTANT: you must specify those pseudo-classes in that order, or changes might not appear the way you want them to. Let’s say I have a black background & I want all my links on the entire page to be yellow all of the time, unless one is being clicked on, in which case I want it to become orange. This is how I would do it (notice that the first selector is a list): 

a:link,
a:visited,
a:hover {
  color: yellow;
}

a:active {
  color: orange;
}

Note that I kept the order listed above, which is LVHA, for :link, :visited, :hover, & :active. There are mnemonics to help you remember this: Licks very hairy arms (that is memorable!), Lazy vultures hopped awkwardly (great mental image!), or the classic LoVe HAte (which is pretty clever, I have to say).

In an effort to reduce the space inside the <figure> between the <p> containing the <img> & the <figcaption> after it, your last assignment had you set margin-top: 0; for the figcaption. However, after making that change, there was still space there! Now let’s fix the problem completely.

To understand why there is still space between <p> containing the <img> & <figcaption>, you need to learn about a CSS concept called margin collapsing. Margin collapsing occurs when two vertical margins — the bottom of one block box, say, and the top of the block box that comes right after it — come into contact & merge into a single margin, taking the larger of the two values.

In other words, if a <p> with margin-bottom: 1.5em; is followed by a <div> with margin-top: 1em;, the distance between the two boxes will not be 2.5em (1.5em + 1em). Instead, since the margins collapse into each other, it will be the larger of the two, so the distance will actually be 1.5em, because that is bigger than 1em (of course, if they are both 1em, they simply collapse into each other so that the combined margin will be 1em!).

This is a very useful layout technique you’ve learned, but there are some big caveats.

  • Only use multi-column layouts when the resulting content will not be very tall. If you did a whole page like that, your reader would get to the bottom of a column & then have to scroll back to the top of the page to continue reading, repeating this for each column. This would very quickly get annoying. Our list is so short that a usability issue like that won’t be possible.
  • That brings us to mobile. Obviously this layout would suck on phones! That’s why we would use responsive layout to turn this layout off for phones & maybe even tablets too.
  • Finally, accessibility. Multi-column layouts can present challenges for users with visual impairments or those using assistive technologies like screen readers, so keep that in mind.
