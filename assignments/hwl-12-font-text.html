In this assignment you will use these new CSS properties:

  • border-bottom & border-top: We’ve previously used border-left to create a borders on the left of a blockquote. Now we will use border-bottom & border-top to create borders on the bottom & top of an element (this should seem kinda obvious!).
  • float: Specifies that an element should be taken from the normal flow (which refers to the way elements are positioned by default in the webpage based on their order in the HTML code) & instead positioned to the left or right inside its container, with the rest of the content flowing around it. You can only float in one direction, either left or right, & that is it.
  • font-weight: Specifies the weight, or boldness, of a font. The values can be either keywords (normal, bold, bolder, & lighter; of these, I find bolder & lighter to be pretty vague & not very useful) or numeric. The numbers range from 100 to 900, incrementing by 100 each time (so 100, 200, 300, all the way up to 900). 400 is equivalent to normal, which is the default weight. 700 is equivalent to bold. 100 is also called thin & is very light, while 900 is also called black & is very heavy.
  • line-height: Specifies the vertical space between lines of text within a block-level element. The value should be a number without units, like 1.3. 1.3 would mean 1.3 times the total height of the font, which is measured from the bottom descender to the top ascender.
  • text-transform: Specifies capitalization of text. There are 4 values: capitalize (Every Word), uppercase (ALL CAPS), lowercase, & none (do not change the current capitalization).

seasonal.html

Several assignments ago I had you add an <aside> to seasonal.html, but we never styled it. Let’s fix that now.

We use <aside> for two purposes on this website:

  • As a landmark indicating an area we will turn into a sidebar using CSS
  • As a pull quote, a quotation that is pulled out of the text & featured

We created a landmark on seed.html using <aside> so that it can eventually become a sidebar. The <aside> on seasonal.html, however, surrounds text that we want to style as a pull quote.

The problem is that when we styled aside earlier in our CSS, we used that as our selector. Therefore, we will have to add a class to the <aside> on the seasonal.html page that specifically indicates that this <aside> is a pull quote. So how about a class named, uh, pull-quote?

Earlier we styled <aside> to have a background-color of #1D824B (a dark green) & a color of #eee (off-white). We don’t want to do that with our pull quote <aside>. Instead, we want the background of the pull quote to be the same color as the webpage, which in this case is white. Then we’ll re-use the dark green color that is the background color of <aside> and apply it to the color of the text in our pull quote. Clever, I know!

Based on that previous paragraph, you might think that we will do this in main.css: 

.pull-quote {
  background-color: white;
  color: #1D824B;
}

This would work, but what if we change the background color of the webpage to an off-white, or something else? Then our white background would stick out like a billy goat at a ballet, & no one wants that. Instead of white, we’re going to a special CSS value that works with virtually every CSS property: initial.

initial can be really useful. When you apply initial to a CSS property (e.g., background-color, margin, or even something complex like animation), you tell the browser’s rendering engine to apply the initial (default) value for that property. And how can you find out what the default value would be? Use MDN!

Let’s take a look at the MDN page for margin as an example. Visit the MDN page for margin & scroll down past “Try it” & “Constituent properties” & “Syntax” & “Description” until you get to “Formal definition”. The very first thing listed there is “Initial value”, which tells you that the default value for margin-top, margin-right, & so on is 0, that is, no margin at all.

NOTE: “But Scott,” I hear you saying, “if the default for all margins is 0, why do my <p> & <h2> tags all have margins on the top & bottom, & my <blockquote> tags have margins on the top & bottom and on the right & left?”

I will look at you with bemusement, take the toothpick out of my mouth, finish my mouthful of root beer, & then say:

Young Padawan, you are confused by the difference between CSS defaults & Web browser rendering engine defaults. Remember, every Web browser has built-in style sheets that make certain elements look & act a certain way that all browser makers have agreed to*. Those are Web browser rendering engine defaults. So yes, the initial CSS value for all margins is 0, but all Web browsers set a rendering engine default of 1em for margin-top & margin-bottom (equivalent to 16px) to all <p> tags.

<h2> tags, however, have 19.92px margins on top & bottom, as you can see in this image showing the Chromium Inspector focusing on an <h2>.

19.92px? Where did that number come from? Here’s how browsers arrived at that very specific number:

  • 1em is the total height of a font based on the size of the font, so if the font-size is set to 18px, then 1em equals 18px, but if the font-size is set to 22px, then 1em equals 22px, & so on.
  • 1em by default in Web browsers is set to 16px
  • <h2> is set to 1.5em in all browsers
  • 16px × 1.5 equals 24px, which is the default font-size for all <h2>s
  • 24px × .83 is 19.9px, which, lo & behold!, is the size of the top & bottom margins for <h2>!

So that’s <p> & <h2>. But what about <blockquote>? Sure, the CSS defaults for <margin> are 0 on all four sides, but the Web browser defaults for blockquote are: 

blockquote {
  display: block;
  margin-top: 1em;
  margin-bottom: 1em;
  margin-left: 40px;
  margin-right: 40px;
}

So there you go — hopefully now it’s clear to you what the difference is between CSS defaults & Web browser rendering engine defaults.

And at that point, I’ll take another swig of root beer, put the toothpick back in my mouth, & continue on my way.

Back to background-color. If you visit its MDN page, & scroll down to “Formal definition”, you’ll discover that the initial value for background-color isâ€¦ transparent! So, by switching background-color: white to background-color: initial, you are setting the background to be transparent, since that is the default value for background-color. That means our pull quote will always take on the color of the webpage it’s sitting on top of.

Now that we have overridden the styles we early set for <aside>, let’s add more. First of all, let’s add a nice border to the top & bottom of .pull-quote that is .5em wide, solid, & a color of #1D824B. That looks nice.

Sometimes pull quotes take up the full width of its container, but that’s not the case here. Instead, we are going to float the pull quote to the left, which is pretty common. There is one big potential gotcha, however: a floated element must have a width, or things will not go as planned. If you’re floating an image, you don’t need to specify that, because an image already possesses an intrinsic width. But if you’re floating any other element, like a <p> or <div>, you definitely need to specify a width in your CSS.

So how wide should we make this particular pull quote? We don’t want it to be too wide, so let’s add a max-width of 20ch. ch? What’s that? So far we’ve used px & em to set the size of text. ch is a unit of measure that is equal to the width of the number 0 in the font (if the width of the number 0 cannot be determined for some reason, then 1ch is 0.5em wide by 1em tall). Interestingly, it’s the only unit of measure that is equal to the width of the character, not the height.

Now we need to add a margin-right to the pull quote so that it doesn’t run into the text to the right of it. Let’s try .75rem, which should keep it from running into the text, but not so much that it looks ridiculous. We need to take care of the bottom margin, too, so let’s add a margin-bottom of .5rem, which should keep it from running into the text below it. Of course, we want a margin-left of 0, because the pull quote is floated all the way to the left, & we also want a margin-top of 0, because we want the pull quote to be flush against the top of the text it surrounds.

When you specify the margins for all four sides of an element, you do know you can do it in a single declaration with just margin, right? Just remember the order of the values: margin: top right bottom left.

We have the margins all sorted out, but the text is still too close to the edges of the pull quote’s box. Let’s add a padding of .5rem to all four sides, which should be easy.

Finally, let’s specify the typeface used in the pull quote. We’ll use font-family: "Iowan Old Style","Palatino Linotype", "URW Palladio L", P052, "Noto Serif", serif; as our font stack. Why those particular fonts? Iowan Old Style is for all Apple devices, Palatino Linotype is for Windows, URW Palladio L & P052 are both for Linux, Noto Serif is for Android, & serif is for everything else. This font stack focuses on Old Style serif typefaces, which, according to Modern Font Stacks, are “characterized by diagonal stress, low contrast between thick and thin strokes, and rounded serifs, and were developed in the Renaissance period”.

Just two more things, since we’re dealing with the typefece. To make the pull quote stand out more, let’s add a font-weight of 600 & a font-size of 1.1rem, which will make the pull quote a bit darker & a bit larger than the body text. But wait! What if the font that gets applied to the pull quote is not available in a weight of 600? In that case, the rendering engine will apply a weight of 700 to the pull quote, because that’s the next available weight that is closest to 600.
