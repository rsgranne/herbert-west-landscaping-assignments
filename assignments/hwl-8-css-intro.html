Terms & definitions

I’m going to have you type some things that won’t really make complete sense until the next assignment. However, you will still be able to start picking up how CSS works just by trying out my instructions.

Look at the following example that you might find in main.css (but don’t type it in there — this is just an example!): 

p {
  color: red; 
}

This whole thing, from p to the final }, is a CSS style rule. Let’s break down the components of that style rule so you know what I am referencing below.

The p is a CSS selector. A selector in the CSS should match something in the DOM. There are many different kinds of selectors, & in this case, the p is a type selector. Any time the selector is an HTML element, that is a type selector. So here, I am telling the browser’s rendering engine to look for any <p> elements in the DOM & do something to them.

OK, so what do I want done to p, my type selector? To find that out, the browser’s rendering engine looks at the style rule’s declaration block, which consists of everything inside the { and } (those are called curly braces). Each line in the declaration block is called a declaration, and declarations always end with a ; (semicolon). In this case, we only have one declaration inside our declaration block: color: red;.

color is a property, while red is a value, so a declaration always consists of a property & a value, separated by a : (colon) and ending with a ;.

A property is the aspect of the selector that you want to change. In this case, we want to change the color of the text, so we use the color property (not all properties are that simple!). A value tells the rendering engine how we want to change that aspect of the selector. In this case, we want the rendering engine to change the color of all <p> elements to red.

And yes, in this case, we would change the text inside every <p> to red. This would look absolutely terrible, so we are not actually going to do that — but you get the idea.

Your first CSS

Please note that we are going to use CSS to change our webpages, but things are still going to be pretty ugly for a while!

In this assignment you will use 3 different CSS properties:

  • background-color: Changes the background color of DOM boxes. There are many ways to express colors.
  • color: Changes the color of text. When people created this property, they made a mistake: they should have named it text-color, which would have made its purpose far more clear.
  • font-family: Tells the rendering engine which typefaces you want to use, in order from most desired to generic fallback, e.g., font-family: Verdana, Arial, Helvetica, sans-serif;. The rendering engine will try to use the first one in the list, but if it can’t find that typeface, it will move to the second one. If the rendering engine can use that typeface, it will; if it cannot, it will move to the next one in the list, and so on, until it finally reaches the generic family name, which can be either serif, sans-serif, monospace, cursive, or fantasy.

html

Given what you know above, let’s start at the top of main.css by changing the typeface (the font-family) for the html selector. The font-family you should use will be:

"Source Sans 3", Verdana, Geneva, Tahoma, sans-serif

Why are there quotation marks around Source Sans 3? Because there are multiple words in the typeface name. In other words, you would put quotation marks around Times New Roman (like this: "Times New Roman") but not around Times (& yes, those are different typefaces).

A list of fonts like this is called a font stack. We could have just provided one font only, like font-family: "Source Sans 3", but what if a user didn’t have Source Sans 3 (& most people would not)? OK, so we’ll add Verdana to the list, creating a very short stack of fonts: font-family: "Source Sans 3", Verdana. If the user doesn’t have Source Sans 3, then use Verdana. But wait — what if the visitor doesn’t have Source Sans 3 or Verdana? Then we tell their browser to use Geneva: font-family: "Source Sans 3", Verdana, Geneva. But wait…

Now you can hopefully see the purpose of a font stack: you specify the font you really want to use first (Source Sans 3 in this case), & then you specify a fallback font, which here is Verdana. Then you specify other fallback fonts, in the order they should be used if the webpage visitor is missing any previously-listed fonts. Eventually you end with a generic font family, which is your way of saying, “OK, if you don’t have any of the fonts in this list, just use whatever default you have for this generic font family. There are five generic font families you can use: serif, sans-serif, monospace, cursive (handwriting fonts), & fantasy (anything that doesn’t fit into the previous four categories).

As soon as you hit Save in VS Code, you should notice a difference when you view a webpage using Live Preview; namely, that the rendering engine is now using a different typeface, probably Verdana, to display the webpage. Headings

Now let’s set a different font-family for headings. It’s a common design trope to use one style of typeface for the body text & a contrasting style for headings. In our case, we used a sans-serif typeface for the body & a serif typeface for the headings (don’t worry if you’re not sure what that means — we’ll cover it later when we get to Typography).

Our selector will consist of every heading, even those we haven’t used yet, because you never know. To apply the same declaration block to multiple selectors, you use a selector list, like this:

h1, h2, h3, h4, h5, h6

The font-family we’ll use with this selector is:

"Source Serif 4", "Iowan Old Style", Constantia, Georgia, serif

Save main.css & take a look at your webpage. Now you should see a different typeface for all the headings.
