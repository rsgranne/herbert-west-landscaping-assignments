In this assignment you will use these new CSS properties, sorted into categories:

  • Typography
    • font-size: Changes the size of text. The default font-size is 16px. You can size text in a variety of methods, including px (pixels), em (1em is equivalent to the line-height of the current font, so if the current font is 18px, 1em would be equivalent to 18px), & rem (1rem is equivalent to the font-size for html (rem stands for root em, because html is the root element), which defaults to 16px). For more on the sizing units you can use with font-size, look for the <length> section in the CSS - Data Types presentation.
    • font-style: Lets you italicize text. For more on this property, look for the font-style section in the CSS - Typography presentation.
  • Lists: list-style-type: Makes it possible to change the marker (e.g., the bullet or number, or a large variety of other choices) in front of list items. In addition, you can use this property to remove the marker so that list items have nothing in front of them. For more on this property, look for the list-style-type section in the Tables & Lists: Organized Data presentation.
  • Layout
    • display: This is a very complex property that allows you set whether an element generates a block or inline box; further, it can determine how the element’s children lay out. There are 18 different possible values for display, and they each do something slightly different, hence the complexity. For more on this property, look for the display section in the CSS Layout: The Visual Formatting Model presentation.
    • width: Obviously sets the width of an element. For more on this property, look for the “Width & Height” section in the CSS Layout: The Visual Formatting Model presentation.
  • Box Model (to read more about all the properties in this section, look for the “Box Model” section in the CSS Layout: The Visual Formatting Model presentation)
    • border: Sets the border size, style, & width for all 4 sides of an element.
    • border-left: Sets the left border’s size, style, & width of an element.
    • border-radius: Rounds the corners of borders.
    • margin: Sets the size of the margin (the area outside the border) for all 4 sides of an element.
    • margin-left: Sets the size of the left margin for an element.
    • margin-right: Sets the size of the right margin for an element.
    • padding: Sets the size of the padding (the area inside the border) for all 4 sides of an element.
    • padding-left: Sets the size of the left padding for an element.

border-left-width: .5rem;
border-left-style: solid;

However, instead of writing all 3 declarations like that, you can use CSS shorthand. Only certain groups of properties can be reduced by using shorthand, & fortunately, this is a great example. Instead of writing all 3 declarations, we can reduce the property to border-left, & then list the width, style, & color. The order doesn’t matter at all. The results would look like this: 

border-left: .5rem solid #ccc;

.callout

Let’s style the Edging heading & the paragraph under it. To do that, wrap both the heading and the paragraph inside a <div>. Apply a class to the <div> named callout.

In main.css, add a new class named callout. Let’s make several changes. To make sure things are working, open landscape-maintenance.html in Live Preview & check it as you add your CSS.

First of all, set the background-color to #313D2B, which is a very dark green. Of course, now you can’t read the text, so let’s change the color to #eee. While we’re at it, let’s set the font-size to 1.25rem. Check your work!

The text is way too close to the edge of the background. To fix that, let’s set the padding. To start out with, try this: 

padding: 1rem;

Go take a look in your Web browser. The left & right sides look good, but there’s too much space at the top & bottom. We can set all 4 sides of padding by using 4 values, representing, in order, the top, right, bottom, & left (TRBL, which sounds like “trouble”, which is how I remember the order), like this: 

padding: 0 1rem 0 1rem;

The 1st value, 0, removes the padding from the top of the box. The 2nd value, 1rem, sets the right side. The 3rd value, 0, removes the padding from the bottom of the box. The 4th value, 1rem, sets the left side.

However, we can use shorthand here (again!) by using 2 values instead of 4, like this: 

padding: 0 1rem;

In this case, the 1st value, 0, removes the padding from the top and bottom of the box (the T and the B of TRBL). The 2nd value, 1rem, sets the padding for the right and left sides of the box (the R and the L of TRBL).

What about a border for our box? Let’s use shorthand again for the property, as we did above, but this time let’s set the value to 2px width, solid style, & #ccc for the color. Take a look in your browser to make sure it’s all working.

Let’s give the corners of that border a little pizzazz by rounding them with the border-radius property. For the value, try .2rem, then change it to .4rem, & then change it to .6rem, where it will remain. After each change, see what the value looks like in the Web browser.

Now let’s set the width of the <div class="callout"> to 80%. That’s nice, but the box is aligned to the left, which looks pretty silly. To fix that, set margin-left and margin-right to auto. This tells the browser to divide the space evenly around the <div>, which results in it being horizontally centered. Nice!
